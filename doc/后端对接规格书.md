# 饱了么 · 后端对接规格书（SSM 版）

> 本文档面向后端开发成员，整合自原始需求文档与前端 UniApp 实际调用情况，以接口为核心组织，去除冗余配置说明，聚焦"后端需要实现什么、前端期望收到什么"。

---

## 一、技术栈与部署约定

| 项目 | 规格 |
|------|------|
| 框架 | Spring 5.x + Spring MVC 5.x + MyBatis 3.x（传统 SSM，非 Spring Boot） |
| 容器 | Apache Tomcat 9+ |
| 数据库 | MySQL 8.x，数据库名 `baoleme_db` |
| 打包方式 | Maven WAR 包，部署到 Tomcat `webapps/` |
| 基础路径 | 所有接口统一前缀 `/api`，Controller 标注 `@RequestMapping("/api")` |
| 跨域 | `spring-mvc.xml` 中配置 CORS，允许前端 UniApp H5 域名及 `*` |
| 字符编码 | `web.xml` 配置 `CharacterEncodingFilter`，强制 UTF-8 |
| 响应格式 | 统一 `application/json`，Controller 标注 `@RestController` |

## 二、数据库 DDL

```sql
CREATE DATABASE IF NOT EXISTS `baoleme_db` CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
USE `baoleme_db`;

-- 用户画像表
CREATE TABLE `user_profile` (
  `id`                   VARCHAR(64)  NOT NULL COMMENT '用户ID / 账号名',
  `name`                 VARCHAR(50)  NOT NULL COMMENT '姓名/昵称',
  `tags`                 VARCHAR(255) DEFAULT '[]' COMMENT '标签JSON数组',
  `history_orders_count` INT          DEFAULT 0,
  `favorite_category`    VARCHAR(50)  DEFAULT '热销推荐',
  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 菜品分类表
CREATE TABLE `category` (
  `id`   VARCHAR(64) NOT NULL,
  `name` VARCHAR(50) NOT NULL,
  `icon` VARCHAR(50) DEFAULT 'Utensils' COMMENT '前端图标标识',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_name` (`name`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 菜品表
CREATE TABLE `dish` (
  `id`          VARCHAR(64)    NOT NULL,
  `name`        VARCHAR(100)   NOT NULL,
  `price`       DECIMAL(10,2)  NOT NULL,
  `description` TEXT,
  `image`       VARCHAR(500)   DEFAULT '',
  `category`    VARCHAR(50)    NOT NULL COMMENT '关联 category.name',
  `sales`       INT            DEFAULT 0,
  `stock`       INT            DEFAULT 99,
  `status`      VARCHAR(20)    DEFAULT 'active' COMMENT 'active | inactive',
  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 订单主表
CREATE TABLE `orders` (
  `id`          VARCHAR(64)   NOT NULL,
  `user_id`     VARCHAR(64)   NOT NULL,
  `total_price` DECIMAL(10,2) NOT NULL,
  `status`      VARCHAR(30)   NOT NULL DEFAULT 'pending'
                COMMENT 'pending | accepted | cooking | delivering | completed | cancelled',
  `created_at`  VARCHAR(50)   NOT NULL COMMENT 'ISO 8601 时间戳',
  `address`     VARCHAR(255)  NOT NULL,
  `phone`       VARCHAR(32)   NOT NULL,
  `note`        VARCHAR(255)  DEFAULT '',
  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 订单明细表
CREATE TABLE `order_item` (
  `id`        BIGINT       AUTO_INCREMENT,
  `order_id`  VARCHAR(64)  NOT NULL,
  `dish_id`   VARCHAR(64)  NOT NULL,
  `dish_name` VARCHAR(100) NOT NULL COMMENT '价格快照',
  `price`     DECIMAL(10,2) NOT NULL,
  `quantity`  INT          NOT NULL,
  PRIMARY KEY (`id`),
  KEY `idx_order_id` (`order_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 客服会话表
CREATE TABLE `chat_session` (
  `id`           VARCHAR(64) NOT NULL COMMENT '通常为 session_{userId}',
  `user_id`      VARCHAR(64) NOT NULL,
  `user_email`   VARCHAR(100) DEFAULT '',
  `mode`         VARCHAR(20)  DEFAULT 'bot' COMMENT 'bot | human',
  `status`       VARCHAR(20)  DEFAULT 'active' COMMENT 'active | resolved',
  `last_updated` VARCHAR(50)  NOT NULL,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_userid` (`user_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 聊天消息表
CREATE TABLE `chat_message` (
  `id`         VARCHAR(64) NOT NULL,
  `session_id` VARCHAR(64) NOT NULL,
  `role`       VARCHAR(20) NOT NULL COMMENT 'user | assistant | merchant',
  `content`    TEXT        NOT NULL,
  `timestamp`  VARCHAR(50) NOT NULL,
  PRIMARY KEY (`id`),
  KEY `idx_session_id` (`session_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 三、接口规格

> **认证说明**：当前前端使用本地账号验证（MOCK），后端可选择实现 Session 或 JWT Token 认证。若实现 Token，前端会在请求头携带 `Authorization: Bearer {token}`。

---

### 3.1 认证接口

#### 用户登录
```
POST /api/auth/login
```
请求体：
```json
{ "username": "xiaoming", "password": "123456" }
```
成功响应 `200`：
```json
{
  "token": "eyJhbGci...",
  "role": "client",
  "profile": {
    "id": "u1",
    "name": "王小明 (加班狂人)",
    "tags": ["熬夜加班", "偏爱重口味"],
    "historyOrdersCount": 24,
    "favoriteCategory": "川湘名菜",
    "phone": "13888888888",
    "address": "北京市中关村科技园A座15层"
  }
}
```
失败响应 `401`：`{ "message": "账号或密码错误" }`

**默认账号：**
- 顾客端：`xiaoming` / `123456`（role: `client`）
- 商家端：`admin` / `admin`（role: `merchant`）

#### 用户注册
```
POST /api/auth/register
```
请求体：
```json
{ "username": "newuser", "password": "123456", "role": "client" }
```
成功响应 `200`：`{ "message": "注册成功" }`
失败响应 `409`：`{ "message": "用户名已存在" }`

### 3.4 订单接口

#### 获取所有订单
```
GET /api/orders
```
响应 `200`（含级联的 items 数组）：
```json
[
  {
    "id": "ord_202605250001",
    "userId": "u1",
    "items": [
      {
        "dish": { "id": "d1", "name": "招牌黄焖鸡米饭", "price": 22.00 },
        "quantity": 2
      }
    ],
    "totalPrice": 44.00,
    "status": "pending",
    "createdAt": "2026-05-25T10:00:00.000Z",
    "address": "北京市中关村科技园A座15层",
    "phone": "13888888888",
    "note": "多加辣"
  }
]
```

> **注意**：前端顾客端会在本地按 `userId` 过滤，商家端展示全量。后端直接返回全量即可。

#### 提交新订单
```
POST /api/orders
```
请求体（与上方响应结构相同）：
```json
{
  "userId": "u1",
  "items": [
    {
      "dish": { "id": "d1", "name": "招牌黄焖鸡米饭", "price": 22.00 },
      "quantity": 2
    }
  ],
  "totalPrice": 44.00,
  "address": "北京市中关村科技园A座15层",
  "phone": "13888888888",
  "note": "多加辣"
}
```
响应 `200`：返回完整订单对象（含后端生成的 `id` 和 `createdAt`）。

**后端必须做的事：**
1. `@Transactional` 保证原子性
2. 扣减库存（防超卖）：`UPDATE dish SET stock = stock - #{qty}, sales = sales + #{qty} WHERE id = #{dishId} AND stock >= #{qty}`，若返回 0 则回滚并返回 `400`
3. 通过 WebSocket 广播新订单通知给所有在线商家（见第五节）

#### 更新订单状态
```
PUT /api/orders/{orderId}/status
```
请求体：`{ "status": "cooking" }`

状态流转：`pending` → `accepted` → `cooking` → `delivering` → `completed`（可直接跳 `cancelled`）

响应 `200`：返回更新后的订单对象。

### 3.5 客服聊天接口

#### 获取或创建用户会话
```
GET /api/chat/session?userId={userId}
```
若该用户无会话则自动创建（`mode: "bot"`, `status: "active"`）。

响应 `200`：
```json
{
  "id": "session_u1",
  "userId": "u1",
  "userEmail": "xiaoming@baoleme.com",
  "mode": "bot",
  "status": "active",
  "lastUpdated": "2026-05-25T10:00:00.000Z"
}
```

#### 获取所有会话（商家端）
```
GET /api/chat/sessions/all
```
响应 `200`：返回所有会话对象数组。

#### 切换客服模式
```
POST /api/chat/session/mode
```
请求体：`{ "sessionId": "session_u1", "mode": "human" }`
响应 `200`：`{ "success": true, "mode": "human" }`

#### 结束会话
```
PUT /api/chat/session/{sessionId}/resolve
```
响应 `200`：`{ "success": true }`

#### 获取会话消息列表
```
GET /api/chat/messages/{sessionId}
```
响应 `200`：
```json
[
  {
    "id": "msg_001",
    "role": "assistant",
    "content": "您好！我是饱了么智能AI客服小饱。",
    "timestamp": "2026-05-25T10:00:00.000Z"
  },
  {
    "id": "msg_002",
    "role": "user",
    "content": "我的订单什么时候到？",
    "timestamp": "2026-05-25T10:01:00.000Z"
  }
]
```

#### 发送消息
```
POST /api/chat/messages
```
请求体：
```json
{
  "sessionId": "session_u1",
  "role": "user",
  "content": "我想催单"
}
```
- `role` 为 `"user"` 时：若当前 `mode` 为 `"bot"`，后端调用 AI 接口生成回复并追加到消息列表；若 `mode` 为 `"human"`，仅存储消息，等待商家回复。
- `role` 为 `"merchant"` 时：直接存储消息。

响应 `200`：
```json
{
  "status": "ok",
  "message": {
    "id": "msg_003",
    "role": "assistant",
    "content": "您的订单正在配送中，预计15分钟内送达。",
    "timestamp": "2026-05-25T10:01:05.000Z"
  }
}
```

## 四、WebSocket 实时推送

### 连接地址
```
ws://{服务器IP}:8080/ws/baoleme/{role}/{userId}
```
- `role`：`client`（顾客）或 `merchant`（商家）
- `userId`：用户 ID

### 消息格式（服务端 → 客户端）

**新订单通知（推送给所有在线商家）：**
```json
{ "type": "NEW_ORDER_PLACED", "payload": { "orderId": "ord_202605250001" } }
```

**订单状态变更（推送给对应顾客）：**
```json
{ "type": "ORDER_STATUS_UPDATED", "payload": { "orderId": "ord_202605250001", "status": "cooking" } }
```

**人工客服消息（推送给顾客）：**
```json
{ "type": "CHAT_MESSAGE", "payload": { "sessionId": "session_u1", "role": "merchant", "content": "您好，请问有什么需要帮助的？" } }
```

### 触发时机

| 事件 | 触发方 | 推送目标 |
|------|--------|----------|
| 顾客下单成功 | `OrderController.createOrder()` | 广播所有在线商家 |
| 商家更新订单状态 | `OrderController.updateStatus()` | 单播对应顾客 |
| 商家发送客服消息 | `ChatController.sendMessage()` | 单播对应顾客 |

## 五、防超卖实现（必须）

在 `DishMapper.java` 中使用原子 SQL，避免并发超卖：

```java
@Update("UPDATE dish SET stock = stock - #{qty}, sales = sales + #{qty} " +
        "WHERE id = #{dishId} AND stock >= #{qty}")
int updateStockAndSales(@Param("dishId") String dishId, @Param("qty") int qty);
```

在 Service 层：
```java
@Transactional(rollbackFor = Exception.class)
public Order createOrder(Order order) {
    for (OrderItem item : order.getItems()) {
        int affected = dishMapper.updateStockAndSales(item.getDishId(), item.getQuantity());
        if (affected == 0) {
            throw new RuntimeException("菜品库存不足: " + item.getDishName());
        }
    }
    // 插入订单主表和明细表...
}
```

## 六、CORS 配置（必须，否则 UniApp H5 无法请求）

在 `spring-mvc.xml` 中添加：

```xml
<mvc:cors>
    <mvc:mapping path="/api/**"
                 allowed-origins="*"
                 allowed-methods="GET,POST,PUT,DELETE,OPTIONS"
                 allowed-headers="*"
                 allow-credentials="false"
                 max-age="3600" />
</mvc:cors>
```
